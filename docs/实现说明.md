## 二、构建与运行环境

- **JDK 13**：使用参考项目随附的 `RustedWarfare/jvm64`，由 `code/use-java13.ps1` 设置会话级 `JAVA_HOME`。
- 源码语法限定 Java 13：使用经典 `switch`，不用 record / switch 表达式。
- **无 Maven、无外部依赖**：JSON 由手写解析器 `data/Json.java` 处理。
- 构建脚本（均为 UTF-8 with BOM，否则 PS 5.1 按 GBK 解析导致中文注释报错）：
  - `compile.ps1`：`javac` 编译 `src/main/java` → `out/`。
  - `build.ps1`：编译 → 打包 `dist/GDUWS.jar`（Main-Class `com.gduws.Main`）→ 用系统 `csc.exe` 编译 `launcher/Launcher.cs` 生成原生 `dist/GDUWS.exe` → 拷贝 `data/`、`assets/` 与内置 JRE（`jvm64`）到 `dist/runtime/`。`-Clean` 强制重拷 JRE。
  - `run.ps1`：开发期快速运行（`compile.ps1` + `java -cp out`）。
- 之所以用 csc 原生 stub + 内置 JRE 而非 jpackage：JDK 13 无 jpackage、无 jmods、无联网。

## 四、与详细设计的关键差异（增量演进）

实现过程中相对最初详细设计做了如下调整，设计文档已同步修订：

1. **角色 `UnitRole` 由静态移到运行时**
   - 设计初稿把 `role` 放在 `UnitDef`（每种单位固定角色）。
   - 实现改为放在 `Unit.role`，由关卡 `enemyUnits[].role` 或布兵阶段按实例指定。
   - 因此**不存在独立的 `scout` 单位类型**；任意单位都能被赋予 `SCOUT` 角色，同一种坦克可同时作打击与侦察。

2. **单位 JSON 不含 `role` 字段**
   - `data/units/*.json` 仅描述静态属性；角色在关卡配置里指定（`level_01.json` 的每个敌方单位带 `role`）。

3. **界面采用 CardLayout 而非独立 Screen 类**
   - 设计列出的 `LevelSelectScreen/DeployScreen/BattleScreen/ResultScreen` 实际由 `GameFrame` 的 `CardLayout` 侧栏（SELECT / DEPLOY / RESULT 卡片）统一承载，`GamePanel` 负责战场绘制。

> 重构提要（低耦合高内聚）：`DeployController`（纯布兵逻辑，无 Swing 依赖）由 `view` 归入 `control`；敌方单位装配从 `GameFrame` 抽到 `control/BattleSetup`，使视图层不再承担模型构建；侦察探索区块跟踪从 `World` 抽到 `model/ExplorationMap`，`World` 仅持有并委派；地形"最近可通行格"查询 `findNearestPassable` 下沉到 `GameMap`。

4. **构建链固化为 PowerShell 脚本**
   - 设计中"Maven / javac 均可"最终落地为三段 PS 脚本 + 原生 exe + 内置 JRE。

5. **地图引擎 v2（分层 + 纹理 + 地块属性）**
   - `TerrainType` 升级为带通行类别 `Pass{LAND,WATER,BLOCK}` 的富枚举：`GRASS/DIRT/SAND`（陆）、`MOUNTAIN`（障碍）、`SHALLOW/WATER/DEEP`（水）；`model` 仍与 AWT 解耦，配色与纹理放在 `view`
   - `Tile` 新增 `decoration`（树/灌木/岩石，纯表现）与 `deployable`（禁布区）两个属性；`GameMap` 增 `isDeployable/isDeployForbidden`
   - `.map` 改为分层格式：首个非注释行 `cols rows tileSize`，其后 `[terrain]`/`[decoration]`/`[deploy]` 三层（旧单层地图仍兼容）。地形字符 `.`草地 `,`泥地 `s`沙地 `#`山地 `_`浅水 `~`水域 `=`深水；装饰 `T/t`树 `b`灌木 `R`岩石；布兵 `X`禁布
   - `view/TerrainTextures` 从复用自 RustedWarfare 的图块集（`assets/tilesets/`）裁切并缓存 20×20 子图，缺素材时回退纯色块；`GameRenderer` 按"纹理→装饰→禁布区蒙版→网格"绘制
   - 新增关卡 `level_07 三栖防线`（40×24）展示多地形、装饰与禁布区

## 五、核心运行时数据流

`World.tick()` 单帧流水线（仅 `BATTLE` 态推进）：

```
vision(VisionSystem) → markRegions → ai(AISystem) → movement(MovementSystem)
  → combat(CombatSystem) → removeDead → checkVictory(损失≥90%)
```

- **侦察 FSM（`ScoutAI`）**：全程 `SCOUTING`，按区块最旧访问时间经 `World.exploration().pickGoal`（`ExplorationMap`，REGION_SIZE=4，每 tick 由友方位置更新访问表）选目标，`findPath(avoidEnemies=true)` 绕开已知敌人。
- **打击 FSM（`StrikeAI`）**：`IDLE → MOVING_TO_TARGET → ATTACKING`；半径内敌方 HP > 友方 HP / RETREAT_RATIO(0.5) 时 `RETREATING`。
- **情报**：`World` 持有 `EnumMap<Faction, IntelBoard>`；打击单位查询本阵营 `IntelBoard` 而非自身视野，实现"侦察发现后打击才行动"。
- **战斗**：`CombatSystem` 选射程内、`AttackProfile.canTarget()` 通过的最近目标，瞬时扣血，`recentShots` 供渲染攻击连线。
- **胜负**：`World.startBattle()` 记录各阵营初始数为基线，`lossRatio >= 0.9` 触发结束；`reset()` 支持重玩。

渲染：`GameRenderer` 在单位上方画血条，按 `recentShots` 画攻击连线（蓝=玩家、橙=敌方）；`GameLoop` 的 `setOnVictory` 回调在胜负产生时自动停表。
