## 二、构建与运行环境

- **JDK 17+**（Temurin 等完整 JDK，需要 `jlink` 和 `jmods`）。
- **无 Maven、无外部依赖**：JSON 由手写解析器 `data/Json.java` 处理。仅音频解码依赖 `libs/` 下的 `jorbis`/`jogg` 两个 JAR。
- 构建脚本（PowerShell）：
  - `compile.ps1`：`javac -encoding UTF-8` 编译 `src/main/java` → `out/`。
  - `build.ps1`：编译 → 打包 `dist/GDUWS.jar`（Main-Class `com.gduws.Main`，第三方 JAR 解包并入）→ 用系统 `csc.exe` 编译 `launcher/Launcher.cs` 生成原生 `dist/GDUWS.exe` → `jlink` 生成最小运行时到 `dist/runtime/` → 拷贝 `data/`、`assets/`。`-Clean` 强制重建运行时。
  - `run.ps1`：开发期快速运行（编译 + `java -cp "out;libs/*" com.gduws.Main`）。
- 产物 `code/dist/` 为自包含发行包，双击 `GDUWS.exe` 即可运行。

## 四、与概要设计的关键差异

实现过程中相对概要设计做了如下调整：

1. **角色 `UnitRole` 是运行时属性**
   - 放在 `Unit.role` 而非 `UnitDef`。由关卡配置（`enemyUnits[].role`）或布兵阶段按实例指定。
   - 任意单位类型都可被赋予 `SCOUT` 或 `STRIKE` 角色，不存在独立的"侦察单位"类型。

2. **界面采用 CardLayout 而非独立 Screen 类**
   - `GameFrame` 用 `CardLayout` 侧栏（SELECT / DEPLOY / RESULT 卡片）统一承载四个界面状态。`GamePanel` 负责战场绘制。

3. **地图引擎 v2（分层格式 + 纹理 + 地块属性）**
   - `TerrainType` 含 7 种地形：`GRASS/DIRT/SAND/MOUNTAIN/SHALLOW/WATER/DEEP`，每种带 `Pass{LAND,WATER,BLOCK}` 通行类别。
   - `Tile` 含 `terrain`、`decoration`（树/灌木/岩石，纯表现）、`deployable`（布兵允许标记）三个属性。
   - `.map` 文件为分层格式：首行 `cols rows tileSize`，其后 `[terrain]`/`[decoration]`/`[deploy]` 三层字符网格。

4. **构建链固化为 PS 脚本 + jlink + 原生 exe**
   - 无 Maven/Gradle，三段 PowerShell 脚本完成编译/运行/打包全流程。

5. **重构要点（低耦合高内聚）**
   - `DeployController`（纯布兵逻辑，无 Swing 依赖）归入 `control` 包。
   - 敌方单位装配从 `GameFrame` 抽到 `control/BattleSetup`。
   - 侦察探索区块跟踪从 `World` 抽到 `model/ExplorationMap`。
   - 地形"最近可通行格"查询 `findNearestPassable` 下沉到 `GameMap`。

## 五、核心运行时数据流

`World.tick()` 单帧流水线（仅 `BATTLE` 态由 `GameLoop` 以 30 tick/s 推进）：

```
VisionSystem → markFriendlyRegions → AISystem → MovementSystem → CombatSystem → removeDead → checkVictory
```

- **侦察 FSM（`ScoutAI`）**：全程 `SCOUTING`，由 `ExplorationMap.pickGoal()` 选最久未访问区块（REGION_SIZE=4），`findPath(avoidEnemies=true)` 绕开已知敌人。机会主义：周边友方兵力 ≥ 敌方×1.8 时主动出击。
- **打击 FSM（`StrikeAI`）**：`IDLE → MOVING_TO_TARGET → ATTACKING`；半径内敌方 HP > 友方 HP × 2 时触发 `RETREATING`。打击单位闲置超过 240 tick 自动转为侦察（`Unit.autoScoutFromStrike` 标记），主动外出寻敌；一旦本阵营情报板出现已知敌人，这些被自动转换的单位立即转回打击投入战斗，避免场上只剩侦察单位。
- **情报共享**：`World` 持有 `EnumMap<Faction, IntelBoard>`；`VisionSystem` 将视野内敌人写入 IntelBoard，打击单位查询 IntelBoard 而非自身视野选目标，实现"侦察发现 → 共享情报 → 打击行动"。
- **战斗**：`CombatSystem` 选射程内、`AttackProfile.canTarget()` 通过（四维攻击域）的最近目标，瞬时扣血。`recentShots` 供渲染攻击连线。
- **胜负判定**：某方损失 ≥90% 立即结算；若双方 900 tick（约 30 秒）无兵力损失，按损失率判定胜负（损失率高者负，相等时玩家胜）。`reset()` 支持完整重玩。

## 六、数据文件格式

### 单位配置 `data/units/*.json`

```json
{
  "id": "light_tank",
  "displayName": "轻型坦克",
  "maxHp": 210,
  "radius": 11,
  "movementType": "LAND",
  "moveSpeed": 1.1,
  "sightRange": 120,
  "spritePath": "assets/units/tanks/tank.png",
  "turretSpritePath": "assets/units/tanks/tank_turret.png",
  "attack": {
    "canAttackLand": true,
    "canAttackWaterSurface": true,
    "canAttackAir": false,
    "canAttackUnderwater": false,
    "maxAttackRange": 130,
    "directDamage": 25,
    "shootDelay": 75
  }
}
```

> 注意：`role`（侦察/打击）**不在单位定义中**，而是在关卡配置或布兵阶段按实例指定。

### 关卡配置 `data/levels/*.json`

```json
{
  "id": "level_01",
  "name": "草地",
  "map": "data/maps/level_01.map",
  "playerBudget": { "light_tank": 10, "interceptor": 1 },
  "enemyUnits": [
    { "unitId": "light_tank", "col": 45, "row": 5, "role": "SCOUT" },
    { "unitId": "light_tank", "col": 45, "row": 10, "role": "STRIKE" }
  ]
}
```

### 地图格式 `data/maps/*.map`

首行为 `cols rows tileSize`。其后为分层字符网格：`[terrain]`（`.` 草地 `,` 泥地 `s` 沙地 `#` 山地 `_` 浅水 `~` 水域 `=` 深水）、`[decoration]`（`T/t` 树 `b` 灌木 `R` 岩石）、`[deploy]`（`X` 禁布）。各层均可省略。

## 七、渲染要点

- `GameRenderer` 渲染顺序：地形纹理（缺素材时回退纯色块）→ 装饰物 → 禁布区蒙版 → 网格线 → 视野圈 → 路径 → 攻击连线 → 单位精灵 → 选中环 → HP 条 → 已知敌情标记。
- `GamePanel` 提供缩放（鼠标滚轮，以光标为锚点）与平移（右键拖拽），以及框选（左键拖拽）。
- 单位底座与炮塔分开绘制，按各自的朝向（`facing`/`turretFacing`）独立旋转。
- 空中单位绘制地面阴影体现高度层。
- 背景音乐由 `MusicPlayer` 通过 JOrbis 解码 OGG，按场景（MENU/BATTLE）随机循环播放。
