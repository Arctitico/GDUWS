> 本文档记录 **Ghost Domains: Unmanned Warfare Sim（GDUWS）** 当前已实现的实际架构，作为代码与设计文档之间的对照纪实。

## 二、构建与运行环境

- **JDK 17+**（Temurin 等完整 JDK，需要 `jlink` 和 `jmods`）。
- **无 Maven、无外部依赖**：JSON 由手写解析器 `data/Json.java` 处理。仅音频解码依赖 `libs/` 下的 `jorbis`/`jogg` 两个 JAR。
- 构建脚本（PowerShell）：
  - `compile.ps1`：`javac -encoding UTF-8` 编译 `src/main/java` → `out/`。
  - `build.ps1`：编译 → 打包 `dist/GDUWS.jar`（Main-Class `com.gduws.Main`，第三方 JAR 解包并入）→ 用系统 `csc.exe` 编译 `launcher/Launcher.cs` 生成原生 `dist/GDUWS.exe` → `jlink` 生成最小运行时到 `dist/runtime/` → 拷贝 `data/`、`assets/`。`-Clean` 强制重建运行时。
  - `run.ps1`：开发期快速运行（编译 + `java -cp "out;libs/*" com.gduws.Main`）。
- 产物 `code/dist/` 为自包含发行包，双击 `GDUWS.exe` 即可运行。

## 四、与概要设计的关键差异

实现过程中相对概要设计做了如下调整：

1. **角色 `UnitRole` 是运行时属性**
   - 放在 `Unit.role` 而非 `UnitDef`。由关卡配置（`enemyUnits[].role`）或布兵阶段按实例指定。
   - 任意单位类型都可被赋予 `SCOUT` 或 `STRIKE` 角色，不存在独立的"侦察单位"类型。

2. **界面采用 CardLayout 而非独立 Screen 类**
   - `GameFrame` 用 `CardLayout` 侧栏（SELECT / DEPLOY / RESULT 卡片）统一承载四个界面状态。`GamePanel` 负责战场绘制。

3. **地图引擎 v2（分层格式 + 纹理 + 地块属性）**
   - `TerrainType` 含 7 种地形：`GRASS/DIRT/SAND/MOUNTAIN/SHALLOW/WATER/DEEP`，每种带 `Pass{LAND,WATER,BLOCK}` 通行类别。
   - `Tile` 含 `terrain`、`decoration`（树/灌木/岩石，纯表现）、`deployable`（布兵允许标记）三个属性。
   - `.map` 文件为分层格式：首行 `cols rows tileSize`，其后 `[terrain]`/`[decoration]`/`[deploy]` 三层字符网格。

4. **构建链固化为 PS 脚本 + jlink + 原生 exe**
   - 无 Maven/Gradle，三段 PowerShell 脚本完成编译/运行/打包全流程。

5. **重构要点（低耦合高内聚）**
   - `DeployController`（纯布兵逻辑，无 Swing 依赖）归入 `control` 包。
   - 敌方单位装配从 `GameFrame` 抽到 `control/BattleSetup`。
   - 侦察探索区块跟踪从 `World` 抽到 `model/ExplorationMap`。
   - 地形"最近可通行格"查询 `findNearestPassable` 下沉到 `GameMap`。

## 五、核心运行时数据流

`World.tick()` 单帧流水线（仅 `BATTLE` 态由 `GameLoop` 以 30 tick/s 推进）：

```
VisionSystem → markFriendlyRegions → AISystem → MovementSystem → CombatSystem → ProjectileSystem → removeDead → checkVictory
```

- **侦察 FSM（`ScoutAI`）**：全程 `SCOUTING`，由 `ExplorationMap.pickGoal()` 选探索目标区块（REGION_SIZE=4），`findPath(avoidEnemies=true)` 绕开已知敌人。机会主义：周边友方兵力 ≥ 敌方×1.5（`ENGAGE_POWER_RATIO`）时主动出击。
  - **目标选择评分**：`pickGoal` 不再"最旧 tick + 取最远"硬排序（会导致单位在对角线两端来回往返、覆盖不到对角线以外区域），改用加权评分：未探索区优先（`staleness×10`）+ 路远加分（保持"奔远处"风格，`distNorm×3`）− 折返惩罚（`reversePenalty×4`）。`ScoutAI` 以单位底座朝向 `u.facing` 推导行进方向传入，对与当前朝向明显反向（夹角余弦 < 0）的候选区块扣分，从而抑制原路折返、把覆盖范围扩展到对角线以外。
  - **可达性过滤（修复潜艇/水域单位原地不动）**：`pickGoal` 先按单位 `MovementType` 从当前格做 8 邻接洪泛（`computeReachable`，对角线遵循与 `Pathfinder` 一致的"不穿角"规则），得到真正连通的可达格集合；评分**只考虑含可达格的区块**，并直接返回区块内一处可达格作为目标。此前只判断"地形可通行"而非"是否连通可达"，会把不连通水域的目标分配给潜艇/舰船，`findPath` 返回 null → 每 tick 重选同一不可达区块 → 原地冻结。`ScoutAI` 另加兜底：避战寻路意外失败时退化为不避战直达。
- **打击 FSM（`StrikeAI`）**：`IDLE → MOVING_TO_TARGET → ATTACKING`；半径内敌方 HP > 友方 HP × 2 时触发 `RETREATING`。打击单位闲置超过 90 tick（`STRIKE_IDLE_TIMEOUT`）自动转为侦察（`Unit.autoScoutFromStrike` 标记），主动外出寻敌；一旦本阵营情报板出现已知敌人，这些被自动转换的单位立即转回打击投入战斗，避免场上只剩侦察单位。
- **情报共享**：`World` 持有 `EnumMap<Faction, IntelBoard>`；`VisionSystem` 将视野内敌人写入 IntelBoard，打击单位查询 IntelBoard 而非自身视野选目标，实现"侦察发现 → 共享情报 → 打击行动"。
- **战斗**：`CombatSystem` 选射程内、`AttackProfile.canTarget()` 通过（四维攻击域）的最近目标，开火时**发射一枚飞行射弹**（FR-21）而非瞬时扣血；`recentShots` 仅供渲染层画炮口闪光。
- **射弹（FR-21）**：`ProjectileSystem` 每帧推进 `World.projectiles`，到达锁定落点后结算伤害。弹种由 `ProjectileType` 区分：**子弹（BULLET）**飞行快、仅对落点处单一敌方结算 `directDamage`；**炮弹（SHELL）**飞行慢、对落点 `splashRadius` 内全部**敌方**单位按距离线性衰减结算群体伤害（中心满伤、边缘递减至 25%，不误伤友军）。伤害量仍由各单位 `directDamage` 决定，弹种仅决定飞行速度与是否群体。
- **胜负判定**：某方损失 ≥90% 立即结算；若双方 2500 tick（约 83 秒，`STALEMATE_TIMEOUT`）无兵力损失，按损失率判定胜负（损失率高者负，相等时玩家胜）。`reset()` 支持完整重玩。

## 六、数据文件格式

### 单位配置 `data/units/*.json`

```json
{
  "id": "light_tank",
  "displayName": "轻型坦克",
  "maxHp": 210,
  "radius": 11,
  "movementType": "LAND",
  "moveSpeed": 1.1,
  "sightRange": 120,
  "spritePath": "assets/units/tanks/tank.png",
  "turretSpritePath": "assets/units/tanks/tank_turret.png",
  "attack": {
    "canAttackLand": true,
    "canAttackWaterSurface": true,
    "canAttackAir": false,
    "canAttackUnderwater": false,
    "maxAttackRange": 130,
    "directDamage": 25,
    "shootDelay": 75,
    "projectileType": "bullet",
    "projectileSpeed": 9.0
  }
}
```

> 注意：`role`（侦察/打击）**不在单位定义中**，而是在关卡配置或布兵阶段按实例指定。
>
> 弹种字段（FR-21，均可省略）：`projectileType` 取 `bullet`（快速单体，缺省）或 `shell`（慢速群体）；`projectileSpeed` 飞行速度（像素/tick，缺省子弹 8.0 / 炮弹 3.0）；`splashRadius` 群体伤害半径（像素，仅炮弹生效，缺省炮弹 40 / 子弹 0）。当前划分：子弹— light_tank/destroyer/interceptor；炮弹— heavy_tank/battleship/submarine/bomber。

### 关卡配置 `data/levels/*.json`

```json
{
  "id": "level_01",
  "name": "草地",
  "map": "data/maps/level_01.map",
  "playerBudget": { "light_tank": 10, "interceptor": 1 },
  "enemyUnits": [
    { "unitId": "light_tank", "col": 45, "row": 5, "role": "SCOUT" },
    { "unitId": "light_tank", "col": 45, "row": 10, "role": "STRIKE" }
  ]
}
```

### 地图格式 `data/maps/*.map`

首行为 `cols rows tileSize`。其后为分层字符网格：`[terrain]`（`.` 草地 `,` 泥地 `s` 沙地 `#` 山地 `_` 浅水 `~` 水域 `=` 深水）、`[decoration]`（`T/t` 树 `b` 灌木 `R` 岩石）、`[deploy]`（`X` 禁布）。各层均可省略。

## 七、渲染要点

- `GameRenderer` 渲染顺序：地形纹理（缺素材时回退纯色块）→ 装饰物 → 网格线 → 残骸 → 攻击范围圈/路径 → 炮口闪光/飞行射弹/爆炸 → **战争迷雾** → 单位精灵 → 选中环 → HP 条 → 已知敌情标记。
- **射弹渲染（FR-21）** 全程程序绘制（无专用贴图），绘于迷雾层之下故受迷雾遮蔽（视野外的敌方射弹不可见）：子弹为小亮点 + 短拖尾体现高速；炮弹为较大弹体 + 拖尾，落点播放随时间扩散、半径上限为 `splashRadius` 的双层爆炸环。
- **战争迷雾（FR-20）** 由 `FogRenderer` 实现，`GameRenderer.fogMode`（NONE/DEPLOY/BATTLE）控制，布兵/战斗统一使用同一种深蓝黑迷雾色 `RGBA(8,12,24,210)`：
  - DEPLOY 模式：对 `GameMap.isDeployForbidden` 的格子整格铺暗色迷雾（取代原先的红色禁布蒙版，二者已合并为迷雾表现）。
  - BATTLE 模式：先把整张地图铺满迷雾，再用 `AlphaComposite.DstOut` + `RadialGradientPaint` 以己方各单位 `sightRange` 为半径“擦”出柔和的圆形可视窗口（视野并集），边缘渐变过渡。
  - 迷雾绘制到与地图等大的离屏 `BufferedImage` 缓冲（按尺寸缓存复用）后整体贴回战场。
  - **敌方单位隐藏（FR-11）** 由 `hiddenByFog()` 统一裁决：DEPLOY 阶段隐藏全部敌方预置单位，BATTLE 阶段隐藏视野并集之外的敌方单位（连同其 HP 条）。己方单位始终可见。布兵放置仍通过 `world.unitAt` 校验，敌方虽不可见但不会被覆盖放置。
  - 残骸绘制在迷雾层之下被自然遮蔽；情报板已知敌情标记仍可见（属“记忆情报”）。
  - 结算（RESULT）时 `fogMode` 置 NONE，揭开战场全貌。
- `GamePanel` 提供缩放（鼠标滚轮，以光标为锚点）与平移（右键拖拽），以及框选（左键拖拽）。
- 单位底座与炮塔分开绘制，按各自的朝向（`facing`/`turretFacing`）独立旋转。
- 空中单位绘制地面阴影体现高度层。
- 背景音乐由 `MusicPlayer` 通过 JOrbis 解码 OGG，按场景（MENU/BATTLE）随机循环播放。
